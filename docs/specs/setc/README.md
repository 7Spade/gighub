# 📋 SETC - 序列化可執行任務鏈

> **SETC**: Serialized Executable Task Chain  
> **目的**: 定義功能開發的詳細步驟，確保開發一致性和可追蹤性

---

## 🎯 什麼是 SETC?

SETC 是一種任務規格格式，將複雜功能拆解為可執行的任務鏈。每個 SETC 文件包含：

1. **任務目標**: 明確的目標定義
2. **前置條件**: 開始前必須完成的項目
3. **任務清單**: 詳細的步驟分解
4. **驗收標準**: 完成的判斷標準
5. **依賴關係**: 與其他 SETC 的關聯

## 🔄 開發優先級原則

```
帳戶體系 → 藍圖系統 → 任務系統（主核心）→ 其他模組
     │           │            │
     ▼           ▼            ▼
   SETC-01    SETC-01      SETC-02
     │           │            │
     └───────────┴────────────┘
                 │
           每個模組需達到
        「企業標準+生產水平」
           才開始下一個
```

## 📝 SETC 格式規範

每個 SETC 文件必須包含以下章節：

```markdown
# SETC-{編號}: {任務名稱}

## 📋 任務概述
- **優先級**: 🔴 最高 / 🟡 高 / 🟢 中 / 🔵 低
- **預估時間**: X 週
- **依賴**: SETC-XX, SETC-XX

## 🎯 任務目標
清晰描述此任務要達成的目標

## ✅ 前置條件
- [ ] 條件 1
- [ ] 條件 2

## 📌 任務清單

### Phase 1: 子任務名稱
- [ ] 步驟 1
  - 詳細說明
  - 相關檔案：`path/to/file.ts`
- [ ] 步驟 2

### Phase 2: 子任務名稱
- [ ] 步驟 1
- [ ] 步驟 2

## 🧪 驗收標準
- [ ] 標準 1
- [ ] 標準 2

## 📚 相關文檔
- [文檔 1](./path/to/doc.md)
- [文檔 2](./path/to/doc.md)
```

## 🏷️ 狀態標籤

| 標籤 | 說明 |
|------|------|
| 📋 規劃 | 任務已規劃，尚未開始 |
| 🚧 進行中 | 任務開發中 |
| 🔍 審查中 | 代碼審查階段 |
| ✅ 完成 | 任務已完成 |
| ⏸️ 暫停 | 任務暫停 |
| ❌ 取消 | 任務取消 |
